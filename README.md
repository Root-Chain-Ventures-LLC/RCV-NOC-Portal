# RCV NOC Portal

The **RCV NOC Portal** is the hub of the Root Chain Ventures network-operations
platform: it owns identity / single sign-on (SSO), API keys, the module catalog,
and the gateway that fronts the RCV modules — **PingIt**, **Site Look Up**, and
**Outage Track**.

> **Install the Portal first, then add modules from the Portal UI.** The Portal is
> the only standalone install; every module is added through it, and the Portal
> **mints each module's OIDC client + API key and deploys it for you** — you never
> hand-wire module secrets.

## Requirements

- **Docker** path (single host): Docker Engine + Docker Compose v2. Bundles
  Postgres and the gateway for you.
- **Kubernetes** path: Kubernetes 1.24+ with Helm 3.8+, and an **external
  PostgreSQL 14+** (the chart does not bundle a database).
- A DNS name + TLS for a public deploy (the bundled gateway can self-sign for LAN).

All published images and charts are **public** on GHCR — no registry login needed.

## Install with Docker Compose (single host — simplest)

```bash
git clone https://github.com/Root-Chain-Ventures-LLC/RCV-NOC-Portal.git
cd RCV-NOC-Portal
./install.sh
```

`install.sh` generates strong secrets into `.env` (including a random first-login
admin password it prints **once**), pulls the published images, starts the stack
(Portal + Postgres + Caddy gateway), enables one-click module installs, waits for
health, and prints the login URL. Open it, sign in as the admin, and you'll be
forced to set a new password.

- **Pin a release:** set `PORTAL_VERSION=v0.1.0` in `.env` (default `latest`).
- **Production:** set `PORTAL_PUBLIC_BASE_URL=https://noc.yourdomain`, put a real
  cert on the gateway (edit [`caddy/Caddyfile`](caddy/Caddyfile) — swap
  `tls internal` for your email/domain) and set `PORTAL_FORCE_HTTPS_REDIRECT=true`.
- **LAN / dev over HTTP:** set `PORTAL_REQUIRE_HTTPS=false` and
  `PORTAL_PUBLIC_BASE_URL=http://<your-host>`.
- To set config up front, copy `.env.example` to `.env` and edit before running
  `./install.sh` — your values are preserved.

### Common `.env` keys

| Variable | Required | Notes |
|---|---|---|
| `PORTAL_VERSION` | no | Image tag to pull (`latest`, or pin `v0.1.0`). |
| `PORTAL_PUBLIC_BASE_URL` | for SSO / public | External URL of the Portal (OIDC issuer + SSO base). |
| `PORTAL_SECRET_KEY` | yes | Session signing key. Auto-generated by `install.sh`. |
| `PORTAL_SECRETS_ENCRYPTION_KEY` | yes | Fernet key for secrets at rest — must survive restarts. Auto-generated. |
| `POSTGRES_PASSWORD` | yes | Bundled Postgres password. Auto-generated. |
| `PORTAL_DEFAULT_ADMIN_EMAIL` | no | First-run admin email (default `admin@rcv.example.com`). |
| `PORTAL_FORCE_HTTPS_REDIRECT` | no | `true` once you have a real cert on `:443`. |
| `PORTAL_SSO_*` | no | Optional pre-seed of Entra/OIDC SSO (or configure in Settings → Authentication). |

## Install with Kubernetes (Helm)

Charts are published to GHCR OCI — no clone required. Install the Portal with
`rbac.create=true` so it can deploy modules into its own namespace, and use
release name `rcv` so module→Portal URLs resolve:

```bash
PORTAL_SECRET_KEY=$(openssl rand -hex 32)
ENC_KEY=$(python3 -c "import base64,os;print(base64.urlsafe_b64encode(os.urandom(32)).decode())")

helm install rcv oci://ghcr.io/root-chain-ventures-llc/helm/noc-portal --version 0.1.0 \
  --namespace rcv --create-namespace \
  --set rbac.create=true \
  --set secrets.portalSecretKey="$PORTAL_SECRET_KEY" \
  --set secrets.secretsEncryptionKey="$ENC_KEY" \
  --set database.host=<pg-host> --set database.password=<pg-password> \
  --set env.PORTAL_PUBLIC_BASE_URL=https://noc.example.com \
  --set-string env.PORTAL_K8S_MODULE_DB="moduser:modpass@<pg-host>:5432" \
  --set ingress.enabled=true --set ingress.host=noc.example.com
```

`PORTAL_K8S_MODULE_DB` (`user:password@host:5432`) is the cluster Postgres that
in-cluster modules use. See [`values.example.yaml`](values.example.yaml) for a
file-based install. (An umbrella chart
`oci://ghcr.io/root-chain-ventures-llc/helm/rcv-platform` can also bring up the
Portal plus modules declaratively — advanced/GitOps.)

## Add modules

After the Portal is up, sign in as an admin and open **Modules → Install**. The
Portal mints the module's OIDC client + API key and deploys it for you — on
**Docker** via the bundled deployer (enabled automatically by `install.sh`), on
**Kubernetes** in-cluster (via `rbac.create=true`). Module images are public, so
no login is required.

On Docker you can alternatively scaffold a module's stack manually:
`./install.sh add-module pingit` (then register it in the Portal UI).

Module repos: [RCV-Ping-It](https://github.com/Root-Chain-Ventures-LLC/RCV-Ping-It) ·
[RCV-Site-Look-Up](https://github.com/Root-Chain-Ventures-LLC/RCV-Site-Look-Up) ·
[RCV-Outage-Track](https://github.com/Root-Chain-Ventures-LLC/RCV-Outage-Track).

## Update

- **Docker:** `docker compose pull && docker compose up -d` (bump `PORTAL_VERSION`
  first to move between pinned releases). Migrations run automatically.
- **Kubernetes:** `helm upgrade rcv oci://ghcr.io/root-chain-ventures-llc/helm/noc-portal --version <new> …`.

## Uninstall

- **Docker:** `docker compose down` (add `-v` to also delete data volumes).
- **Kubernetes:** `helm uninstall rcv -n rcv`.
